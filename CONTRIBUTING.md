# Contributing to Capitals Quiz

Thank you for your interest in contributing to this project! Here's how you can help.

## 🐛 Reporting Bugs

Found a bug? Please report it by [opening an issue](https://github.com/BodhiProtocol/flashcard-app/issues) with:

- **Clear title** — What's broken?
- **Description** — How to reproduce it?
- **Environment** — Browser, OS, device
- **Expected vs actual** — What should happen vs what does?

### Bug Report Template
```
**Describe the bug:**
A clear description of what the bug is.

**To reproduce:**
Steps to reproduce the behavior:
1. Go to '...'
2. Click on '....'
3. See error

**Expected behavior:**
What should happen?

**Screenshots:**
If applicable, add screenshots.

**Environment:**
- Browser: [e.g. Chrome 120, Firefox 121]
- OS: [e.g. Windows 11, macOS, Linux]
- Device: [e.g. Desktop, Mobile]
```

## 💡 Suggesting Features

Have an idea? [Open a feature request](https://github.com/BodhiProtocol/flashcard-app/issues) with:

- **Clear title** — What feature?
- **Description** — Why is this needed?
- **Use case** — Who benefits?
- **Example** — How would it work?

### Feature Request Template
```
**Describe the feature:**
Clear description of what you want.

**Why is this needed?**
Explain the problem it solves.

**Proposed implementation:**
How should it work?

**Alternative solutions:**
Other approaches you've considered.
```

## 🔧 Development Setup

### Prerequisites
- Text editor (VS Code, Sublime, Notepad++, etc.)
- Git
- Web browser

### Local Setup
```bash
# Clone the repository
git clone https://github.com/BodhiProtocol/flashcard-app.git
cd flashcard-app

# Open the file
# Windows
start capitals-quiz.html

# macOS
open capitals-quiz.html

# Linux
xdg-open capitals-quiz.html
```

### Making Changes
1. Edit `capitals-quiz.html` with your text editor
2. Save the file
3. Refresh your browser to see changes
4. Test thoroughly in different browsers

## 📝 Code Guidelines

### JavaScript Style
- Use **ES6+** syntax
- Keep functions small and focused
- Add comments for complex logic only
- Use meaningful variable names
- No external dependencies

### HTML/CSS
- Keep markup semantic
- Use CSS variables for theming
- Maintain responsive design
- Test on mobile and desktop

### Commenting
```javascript
// Only comment the WHY, not the WHAT
// Example: Handle async loading before data arrives
if (!data) return;
```

## 🚀 Submitting Changes

### 1. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

### 2. Make Your Changes
- Edit files
- Test thoroughly
- Keep commits focused

### 3. Commit with Clear Messages
```bash
git add .
git commit -m "Add feature: description of what you added"
# Example: "Add dark mode support"
```

### 4. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 5. Create a Pull Request
- **Title:** Clear, descriptive title
- **Description:** What does this change?
- **Why:** Why is this change needed?
- **Testing:** How to verify it works?

### Pull Request Template
```markdown
## Description
What does this PR do?

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation update
- [ ] Performance improvement

## Testing
How to test these changes?

## Screenshots
If UI changes, include screenshots.

## Checklist
- [ ] Tested in Chrome
- [ ] Tested in Firefox
- [ ] Tested in Safari
- [ ] Works offline
- [ ] Data persists
```
